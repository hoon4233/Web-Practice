[Location 객체]
	Location 객체는 문서의 주소와 관련된 객체
		이용해서 윈도우의 문서 url을 변경 가능하고 문서의 위치 등 다양한 정보 얻을 수 있음

	url 정보 알아내기 == location.href
	url의 프로토콜 == location.protocol
	url의 호스트 == location.host 
		(호스트란? 서버를, 서비스를 식별하는 주소. 그게 돌아가고 있는 컴퓨터 식별)
	url의 포트 == location.port 
		(포트란? 서버 컴퓨터에서 돌아가는 여러 서버들 중 하나를 고르겠다는 뜻)
	url의 패스 == location.pathname
		(웹에 접속했을때 구체적인 정보를 요청했을때 그 주소)
	url에 전달된 값 == location.search
		( ex)?id == 10 )
	bookmark == location.hash
		(사이트의 특정 위치에 북마크 할 수 있음)

	웹주소는 보통
	protocol + host + port + pathname + search로 구하는값 + hash로 구하는값


	location 하면 url을 변경할 수 있음
	location.href = "www.naver.com" 하면 현재 창이 네이버로 이동함
		다른 url로 이동해야 할 경우에 이렇게 쓰면 될듯
		(location = "www.naver.com"도 가능. 위의 방법이 더 보편적)
	location.href = location.href 하면 리로드 하는 방법
		(location.reload도 가능 이게 더 보편적)



[Navigator 객체]
	navigator
		자바스크립트를 이용해서 현재 자바스크립트가 실행되고 있는 제품의 이름과 버전을 알수 있음

	cross browsing ?
		세상에는 많은 종류의 브라우저가 있음
		브라우저의 동작방법은 w3c에 규정되어 있음
		하지만 아주 디테일한 부분은 각각 브라우저마다 다름
		내가 짠 코드가 다 다르게 작동할 수도
		특성에 맞는 코딩 필요
		->브라우저 마다 다르게 동작한다는 이슈가 cross browsing

	(1)자바스크립트는 네스케이프에서 만든 언어임.
		자사의 브라우저의 성능을 높이기 위해서
	(2)MS가 ie를 만듬. 자사의 OS에 넣어서 팔아버림
	네스케이프와 MS가 기능경쟁을 시작하며 점점 호환되는 부분이 적어짐
		-> 똑같은 동작을 하더라도 함수 이름이 달라짐
	=> 웹 표준을 만들어서 최소화 시킴

	웹표준이 널리 퍼지기 전에 네비게이터 진짜 필수였다고 함
	지금도 중요함


	console.dir == 객체가 가지고 있는 프로퍼티를 모두 알 수 있음
	navigator의 프로퍼티
		appName == Netscape 계열이라는 것(어떤 브라우저인지)
		appVersion == 브라우저의 자세한 정보
		userAgennt == 웹 브라우저를 다르게 이야기 할때 useragent라 부름
						브라우저가 서버에게 요청하니까 agent라 부름
						서버에게 요청할때 보내는 정보 중 하나가 이거
		platform == 지금 사용하는 운영체제를 알려주는 함수

		cross browsing 이슈 해결하는데 도움
		자바스크립트에는 keys라는 메소드가 있음
		최신버전에만 있으므로 기능테스트 필요
		기능테스트 하는법 - 코드 작성해서 만약 그 API가 없으면 만들어주도록


[창 제어]
	자바스크립트를 이용해서 새로운 창을 열고 닫고 하는 것과, 창들끼리의 커뮤니케이션
	window.open. 새로운 텝으로 첫번째 인자로 들어온 파일 열어줌
	두번재 인자로 _self주면 새 텝이 아니라 지금 텝에서 그 파일 열어줌
	두번째 인자로 _blank주면 새 텝으로 그 파일 열어줌
		(아마 open의 두번째 디폴트 값이 _blanck인듯)
	html의 a테그와 같음 (window.open)
	두번째 인자로 ot 주면 첫번째 인자의 이름 텝이 없으면 새로 열리고 만약 있다면 그 창에서 문서가 로드 됨
	세번째 인자에는 크기나 모양(창 크기 설정)등을 정해줄 수 있다



	서버를 설치하고 파일을 올려두고 실습해야함
	window.open의 리턴 값은 새로운 창의 window 객체 값임
	onkeypress 사용하면 키를 누를때마다 = 다음의 함수 실행
		winmessage에 주는 값은 지금 치고 있는 글자들
	아이디 값이 message인 태그의 innertext를 바꿈
	win.close로 새로 연 창 받을 수 있음
	(window.open 할때 win이라는 변수에 그 창을 저장해두었으니)


	보안에 대한 이야기
	브라우저는 우리의 컴퓨터에 접근 할 수 있음

	같은 도메인 위에 있는 파일은 같은 사람이 만들었다고 생각하고 제어할수 있게 해줌( ex) 예제에서 사용한 localhost )
