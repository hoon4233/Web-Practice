[HTML에서 javascript 로드하기]
HTML안에서 자바스크립트 작성하려면 HTML에서 자바스크립트 로드 해야함
1. inline 방식, 이벤트를 위해 자바스크립트를 사용하는 방식
	onclick == html 속성, onclick속성을 가진 태그가 클릭 했을 때 속성의 값으로 들어가 있는 자바스크립트 값을 실행

	onclick은 html
	alert는 자바스크립트

	inlie의 장점 -> 코드 안에 들어가 있어서 뚜렷하게 보임
		   단점 -> 정보와 제어가 섞여있음(유지보수에 어려움)
		   		html(정보)이 필요할때 html찾고 
		   		자바스크립트(제어)필요할때 자바스크립트 찾는게 좋음


2. 스크립트 테그 방식 (<script></script>)
	이전 코드에서 onclick 이라는 코드가 사라짐.
	대신 스크립트테그가 생겼음. 테그 이후에 나오는 문법은 자바스크립트 문법으로 해석
	끝나는 테그 이후에는 다시 html로
	이렇게 하면 html코드 안에 자바스크립트의 코드가 사라질거임

	inline보단 낫지만 그래도 html파일 안에 자바스크립트가 존재하는거임


3. 외부 파일로 분리
	<script src="./script.js"></script>
	장점들
		html파일에서는 무조건 html의 문법만 쓸 수 있게 해줌
			이제 html은 정보만 담게 됨
		다른 html파일에서 똑같은 자바스크립트 파일 사용한다면 이렇게 하는게 효율적
			유지보수의 편의성
		자바스크립트가 캐쉬화 되어 효율 상승될 가능성이 높다
			브라우저는 캐쉬를 가지고 있다.
			이미 다운받은 걸 또 받을 필요 없음
			똑같은 파일을 요청할때는 브라우저가 읽지 않고 저장장치가 읽어줌
				비용 낮아지고 성능 높아짐

4. 스크립트파일의 위치
	스크립트 테그는 헤드에 위치? 바디에 위치 ?
		바디테그가 끝나는 위치에 위치 시키는게 조금 더 바람직함


	헤드테그 안에 위치시킨다면 ?
	html에서 읽다가 스크립트 테그 만나면 자바스크립트 파일 다운로드 하고 나머지 내용 실행. c에서 헤더파일 불러오는거랑 비슷한거 같음
	스크립트 코드 안에서 사용한 변수들(?)이 아직 선언되지 않은 변수들일수 있음
	window.onload = function(){} 안에 함수들을 넣어주면 해결할 수 있긴 함
	브라우저는 웹페이지의 모든 코드가 전부 읽히고 필요한 것들 다 다운로드 끝났을때 윈도우 개체의 onload를 읽도록 되어있음
		(애초에 바디 끝에 스크립트 테그 위치 시키면 필요 없는 함수)
		(성능면에서도 바디 맨 밑에 두는게 더 좋음, 렌더링 문제 등등)



[Object Model]
자바스크립트로 제어 할 수 있도록 브라우저의 구성요소 여러가지를 객체로 만들어줌
BOM (brower object model)

자바스크립트는 객체를 이용해서 제어함
브라우저가 html을 읽으면서 테그에 해당되는 객체를 만들어주고 자바스크립트가 이거를 제어함


DOM은 (document object model)
window(전역객체, window,frame)라는 객체가 있고 document이런거 다 property
window.document라면 window라는 객체의 propery에 들어 있는 document라는 객체
window는 전역객체니까 코드 쓸때 보통 생략 가능
document는 웹페이지의 문서 테그를 제어 한다고 정도만 생각해두기


BOM 역시도 window의 property

javascript core도 window의 property
BOM과 DOM은 웹 브라우저의 객체들을 담고 있음
이런 웹 브라우저의 객체들 말고도 제어해야 할 일들이 생길수 있음(node.js 같은거 쓸때)
어느쪽이든 javascript core에 담겨 있는 객체들을 사용할거임
	(자바스크립트 쓰는건 같으니까)
